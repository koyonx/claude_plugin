# code-convention-learner

プロジェクトのコーディング規約を既存コードから学習し、セッション開始時にスタイルガイドとして注入するプラグイン。

## 機能

- **PostToolUse (Write|Edit)**: 変更ファイルのコーディングパターンを分析・記録
- **SessionStart**: 学習済みの規約をコンテキストに注入

## 分析対象

| 項目 | 対象言語 |
|------|---------|
| インデント (スペース/タブ, 2/4) | 全言語 |
| クォート (シングル/ダブル) | JS/TS, Python |
| セミコロン (あり/なし) | JS/TS |
| 命名規則 (camelCase/snake_case) | 全言語 |
| トレーリングカンマ | JS/TS |

## 対応言語

JavaScript, TypeScript, Python, Go, Rust, Ruby, Java

## 動作条件

- 5行未満のファイルはスキップ
- 1MB超のファイルはスキップ
- 言語あたり100ファイル分析が上限
- 5ファイル以上分析した言語のみ規約を表示

## データ保存

保存先: `~/.claude/code-convention-learner/`
- プロジェクトごとにJSONファイルでパターンカウントを記録
- ファイル内容は一切保存しません
