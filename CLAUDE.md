# Claude Plugin Project

## プロジェクト概要

Claude独自プラグインを開発・管理するリポジトリ。

## ブランチ運用ルール

### 基本方針

- **1プラグイン = 1ブランチ系統** で管理する
- ブランチ名は `plugin/<プラグイン名>` の形式とする（例: `plugin/code-review`）
- プラグインの修正・更新時は、そのプラグインの最後にマージ済みのブランチから新しいブランチを切る
  - 命名例: `plugin/<プラグイン名>-fix-xxx`

### ブランチ作成フロー

1. 新規プラグイン: `main` から `plugin/<プラグイン名>` を作成
2. 既存プラグインの修正: 該当プラグインの最後にマージされたブランチの状態（= `main`の最新）から新ブランチを作成

## PR作成ルール

### レビュープロセス（PR作成前に必須）

1. コード変更が完了したら、**PRを作成する前に**、Codex等の他のAIツールを使用してコードレビューを実施する
2. レビューで指摘された**優先度「高」の修正事項をすべて解消**する
3. 修正後、**再度AIレビューを実施**する
4. **優先度「高」の指摘が0件になるまで、手順2〜3を繰り返す**（レビューループ）
5. 優先度「高」が0件になったことを確認してからPRを作成する
6. 優先度「中」「低」の指摘は対応推奨だが、PR作成のブロッカーとはしない

### PR作成手順

- PRの作成には必ず `gh` コマンドを使用する
- PRタイトルは70文字以内で簡潔に記述する
- PR本文には以下を含める:
  - 変更概要（Summary）
  - テスト計画（Test plan）
  - AIレビュー実施済みの旨

```bash
# PR作成コマンド例
gh pr create --title "Add <plugin-name> plugin" --body "$(cat <<'EOF'
## Summary
- <変更内容を箇条書き>

## Test plan
- [ ] <テスト項目>

## AI Review
- [x] Codex等によるレビュー実施済み
- [x] 優先度「高」の指摘事項は全て解消済み
EOF
)"
```

## 開発ワークフロー

1. `main` から新ブランチを作成
2. プラグインを実装
3. AIレビュー（Codex等）を実施
4. 優先度「高」の指摘を修正
5. 再度AIレビューを実施（優先度「高」が0件になるまで3〜4を繰り返す）
6. `gh pr create` でPRを作成
7. レビュー・マージ
